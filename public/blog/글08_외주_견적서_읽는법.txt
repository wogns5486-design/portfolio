【카테고리】 외주 개발 가이드
【제목】 개발 외주 견적서, 이것만 확인하세요 (체크리스트)

견적서를 받았는데 뭐가 뭔지 모르겠다?
이 글 하나로 견적서 읽는 법을 정리합니다.


■ 견적서에 반드시 있어야 할 항목

1. 작업 범위
→ 어떤 페이지를 몇 개 만드는지
→ 어떤 기능이 포함되는지
→ "홈페이지 제작"만 적혀있으면 위험합니다

2. 가격 (총 비용)
→ 기획, 디자인, 개발, 배포가 전부 포함인지
→ "추가 비용 별도"라는 문구가 있는지 확인

3. 작업 기간
→ 시작일, 완료 예정일
→ 중간 점검 일정이 있는지

4. 수정 횟수
→ 무제한 수정은 현실적으로 불가능
→ 보통 2~5회가 적정

5. 소스코드 제공 여부
→ "제공 안 함"이면 나중에 수정 불가
→ 반드시 확인하세요

6. 유지보수
→ 배포 후 버그 수정은 몇 개월간 무료인지
→ 월 유지보수 비용이 있는지


■ 위험한 견적서 신호

- "상세 내용은 협의 후 안내" → 불투명
- 작업 범위 없이 가격만 적혀있음 → 위험
- 수정 횟수 언급 없음 → 분쟁 소지
- 소스코드 관련 내용 없음 → 물어보세요


■ 좋은 견적서 예시

작업 범위: 메인 페이지, 서비스 소개, 문의 폼 (총 3페이지)
기능: 반응형, 문의 폼 이메일 연동
디자인: 맞춤 디자인 (시안 1차 제공, 수정 2회)
가격: 15만원 (기획+디자인+개발+배포 포함)
기간: 5일
수정: 2회 포함
소스코드: 제공
유지보수: 배포 후 1주 무료 버그 수정


■ 마무리

견적서는 "계약서"와 같습니다.
모호한 부분이 있으면 반드시 물어보세요.
