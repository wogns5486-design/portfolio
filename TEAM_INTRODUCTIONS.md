# 원스텝 AI — 팀 소개

> 11개 부서, 32명의 AI 전문가가 CEO의 능력을 100배로 만들어주는 조직입니다.

---

## 전략기획실 (12명)

### 🏷️ Planner (opus)
> "계획은 전투의 절반입니다"

모든 분석 결과를 통합해서 실행 계획서를 만드는 전략가입니다. CEO님이 방향을 정하면, 저는 그걸 단계별 구현 플레이북으로 바꿔줍니다.

---

### 🏷️ Analyst (opus)
> "수치로 말합니다"

시장과 경쟁사를 깊이 분석하는 데이터 마인입니다. "이 사업이 유망한가?" 같은 질문에 정량 근거로 답합니다. 웹 조사부터 통계 검증까지 전담합니다.

---

### 🏷️ Architect (opus)
> "기술로 비전을 현실화합니다"

CEO님의 아이디어가 기술적으로 실현 가능한지 검토하는 설계자입니다. 기술 제약이 있으면 현실적인 대안을 제시합니다. 스케일, 보안, 통합 관점에서 사업을 봅니다.

---

### 🏷️ Critic (opus)
> "좋은 계획에는 가차없는 검증이 필요합니다"

당신의 가장 엄격한 품질 게이트입니다. 논리 허점, 숨겨진 리스크, 산술 오류, 법적 함정을 찾아냅니다. 제 검증을 통과한 것은 정말 견고합니다.

---

### 🏷️ PM-Lead (opus)
> "한 명이 4명을 지휘합니다"

PM팀의 리드입니다. CEO님이 "이 제품 기획해줘"라고 하면, 4명의 PM을 병렬 투입해서 완성된 PRD까지 만들어줍니다.

**잘하는 것:** 4명 동시 오케스트레이션, 완성도 보장
**이럴 때 부르세요:** "새로운 서비스 기획 필요해" → 3시간 안에 PRD 완성
**차별점:** PM들 각각은 전문가지만, 저는 그들을 하나의 완성된 기획서로 조율합니다.

---

### 🏷️ PM-Discovery (sonnet)
> "기회는 문제 속에 숨어있습니다"

Teresa Torres의 OST 프레임으로 "고객이 정말 원하는 게 뭔가?"를 찾아냅니다.

**잘하는 것:** OST 맵핑, 중요도×(1-만족도) 우선순위 스코어링
**이럴 때 부르세요:** "아이디어는 많은데 뭘 먼저 할지 모르겠어" → 스코어링으로 순서 제시
**차별점:** 다른 PM은 기능부터 생각하는데, 저는 고객의 일(job)부터 시작합니다.

---

### 🏷️ PM-Strategy (sonnet)
> "가치를 전달하는 방식이 제품을 만듭니다"

JTBD 6-Part 가치 제안과 Lean Canvas로 "우리 사업의 핵심 가정"을 명확하게 합니다.

**잘하는 것:** 6-Part 가치 제안 정의, Lean Canvas 9개 섹션 구멍 찾기
**이럴 때 부르세요:** "우리 가치 제안이 뭐야?" → 명확하게 정의
**차별점:** 시장 조사는 외부를 보는 것, 저는 "우리가 이 시장에서 왜 이기는가?"를 정의합니다.

---

### 🏷️ PM-Research (sonnet)
> "데이터가 없으면 추측일 뿐입니다"

페르소나 3개, 경쟁사 5개, TAM/SAM/SOM을 웹 리서치로 검증해서 제공합니다.

**잘하는 것:** 행동 기반 페르소나, Top-Down + Bottom-Up 이중 시장 규모 검증
**이럴 때 부르세요:** "시장이 얼마나 클까?" → TAM/SAM/SOM 수치로 제시
**차별점:** 웹서치 + 통계 검증 + 역산을 한 세트로 합니다.

---

### 🏷️ PM-PRD (sonnet)
> "8섹션이면 완벽한 PRD입니다"

위 세 동료들의 분석을 받아서 8섹션 PRD + Beachhead + GTM을 완성합니다.

**잘하는 것:** Beachhead 세그먼트 선정, GTM 전략, Release Plan
**이럴 때 부르세요:** "모든 분석 나왔으니 PRD로 정리해줘"
**차별점:** PRD는 최종 문서. CEO 결재할 수 있는 형태로 만드는 게 제 일입니다.

---

### 🏷️ Gap-Detector (opus) — ⭐ S등급 (93점)
> "설계와 구현, 그 차이를 찾아냅니다"

PDCA Check 단계 담당. 설계 문서와 구현의 일치율을 정량 점수화합니다.

**잘하는 것:** 5개 카테고리 갭 분석, 내부 90%/클라이언트 95% 동적 임계값, Critical/Warning/Info 심각도 분류
**이럴 때 부르세요:** "설계대로 구현됐나?" → 매칭율 XX.X%로 정량 보고
**차별점:** 다른 QA는 버그를 찾지만, 저는 "설계된 것과 만들어진 것의 일치"를 봅니다.

---

### 🏷️ PDCA-Iterator (sonnet)
> "5번의 기회로 완벽에 가까워집니다"

Gap-Detector가 지적한 갭을 고쳐서 다시 평가하는 루프를 최대 5회 돌립니다.

**잘하는 것:** Critical→Warning→Info 순서로 자동 수정, 매 iteration 전후 점수 추적
**이럴 때 부르세요:** "갭 분석 나왔는데 고쳐줘" → 기준점까지 자동 끌어올림
**차별점:** gap-detector는 "뭐가 빠졌나" 찾고, 저는 "그걸 어떻게 고칠까" 실행합니다.

---

### 🏷️ Brainstormer (opus)
> "뭉뚱그려진 아이디어를 날카로운 기획으로"

"이거 어떻게 하면 좋을까?"라는 막연한 생각을 프로젝트 브리프로 변환합니다.

**잘하는 것:** Mode A(AI끼리 토론), Mode B(CEO 참여 하이브리드), YAGNI 원칙 MVP 스코프 분리
**이럴 때 부르세요:** "이런 기능 추가하고 싶은데 어떨까?" → 30분 안에 브리프 완성
**차별점:** 기획을 하는 게 아니라, CEO의 생각을 기획 가능한 형태로 바꿔줍니다.

---

## 리서치팀 (3명)

### 🏷️ Scientist (sonnet)
> "데이터는 거짓말을 하지 않습니다"

과학적 근거 기반의 정량 분석 전담자입니다. 시장 트렌드 수치화, 통계적 검증, 출처 명시 리서치를 병렬로 투입할 수 있습니다.

**잘하는 것:** 웹 데이터 수집 후 통계 분석, 복수 소스 교차 검증, 수치 기반 리포트 생성
**이럴 때 부르세요:** "이 가설이 통계적으로 유의미한가?" → p-value, 신뢰도, 표본수 분석
**차별점:** 다른 팀원들은 "아마도 그럴 거야"라고 하지만, 저는 "데이터가 이렇게 말해요"라고 합니다.

---

### 🏷️ Document-Specialist (sonnet)
> "공식 문서가 최고의 교과서입니다"

외부 문서와 공식 API 레퍼런스 조사 전문가입니다. SDK, 프레임워크, 라이브러리의 사용법을 문서에서 직접 찾아 정확한 정보를 제공합니다.

**잘하는 것:** 공식 docs/API 레퍼런스 빠른 위치 파악, SDK 버전별 호환성 정리, 공식 예시 제시
**이럴 때 부르세요:** "이 라이브러리 실제로 이렇게 써도 되나?" → 공식 문서로 검증
**차별점:** 추측이나 블로그가 아니라, 공식 문서로만 답합니다.

---

### 🏷️ Explorer (haiku)
> "코드의 지도를 그리는 사람"

대규모 코드베이스를 빠르게 탐색하는 탐험가입니다. 파일 구조, 패턴 검색, 아키텍처 전체상을 순식간에 파악합니다.

**잘하는 것:** Glob/Grep으로 1초 안에 파일 찾기, 폴더 구조 + 의존성 맵핑
**이럴 때 부르세요:** "auth 관련 코드 다 어디있어?" → 3초 내 모든 위치 리스트업
**차별점:** 깊이 파고드는 분석가가 아니라, 전체 그림을 재빠르게 그리는 네비게이터입니다.

---

## 개발팀 (4명)

### 🏷️ Executor (sonnet/opus)
> "오늘도 버그 18개를 잡습니다"

코드 작성과 기능 구현을 전담하는 실행자입니다. 위시켓 3개 프로젝트 18건 버그 수정 완료 실적.

**잘하는 것:** 사양서→구현 코드 변환, 버그 수정/기능 추가/리팩토링, 복잡한 로직은 opus 모드로 처리
**이럴 때 부르세요:** "이 기능 구현해줘" → 테스트 포함, 배포 가능 상태로 완성
**차별점:** "이건 어렵습니다"가 아니라 "이렇게 풀겠습니다"라고 합니다.

---

### 🏷️ Debugger (sonnet)
> "스택 트레이스는 거짓말을 하지 않습니다"

버그의 근본 원인을 파고드는 디버거입니다. 스택 트레이스, 로그, 컴파일 에러를 읽고 "정확히 어디서 왜 터졌는가"를 밝혀냅니다.

**잘하는 것:** 콘솔 에러 → 근본 원인 추적 (5단계 깊이), 빌드 실패 빠른 진단
**이럴 때 부르세요:** "프로덕션에서만 터지는데..." → 환경 차이 분석 + 해결책
**차별점:** executor가 "코드 짜기"라면, 저는 "코드 고장 찾아서 부수기"를 전문으로 합니다.

---

### 🏷️ Code-Simplifier (opus)
> "읽기 쉬운 코드가 유지보수 비용을 줄입니다"

복잡한 코드를 명확하고 단순하게 다듬는 리팩토링 전문가입니다. 기능은 100% 유지하면서 가독성, 일관성, 유지보수성을 극대화합니다.

**잘하는 것:** 스파게티 로직→명확한 흐름 변환, 변수명/함수명/구조 최적화
**이럴 때 부르세요:** "이 함수 너무 길어서 이해 안 돼" (500줄) → 50줄 단위로 분해
**차별점:** executor가 "동작하는 코드"를 만든다면, 저는 "읽을 수 있는 코드"로 변환합니다.

---

### 🏷️ Git-Master (sonnet)
> "히스토리가 말해주는 것을 들어야 합니다"

원자적 커밋, 리베이스, 히스토리 관리를 전담하는 깃 마스터입니다. 프로젝트 스타일을 자동 감지해서 일관된 히스토리를 유지합니다.

**잘하는 것:** 커밋 메시지 컨벤션 자동 감지, 복잡한 병합 상황 해결, 읽을 수 있는 히스토리 구성
**이럴 때 부르세요:** "이 PR 커밋 5개가 뒤섞여 있는데..." → 원자적 커밋으로 정리
**차별점:** 다른 개발자는 `git commit -m "fix"`만 하지만, 저는 히스토리를 설계합니다.

## QA팀 (5명)

### 🏷️ Verifier (sonnet)
> "증거 없는 완료 보고는 없다"

증거 기반의 완료 확인을 담당합니다. 테스트 커버리지, 성능 지표 — 모든 것을 검증 가능한 증거로 확인합니다.

**잘하는 것:** 테스트 충분성 평가, 완료 조건 체크리스트 검증, 결함 재현 가능성 판단
**이럴 때 부르세요:** "이 PR이 정말 준비됐나요?" → 변경사항 vs 테스트 vs 문서 정합성 검토
**차별점:** 텍스트 보고만 믿지 않습니다. 제출된 증거를 직접 검증합니다.

---

### 🏷️ QA-Tester (sonnet)
> "실제 환경에서 직접 테스트한다"

tmux 기반 인터랙티브 CLI 환경에서 실제 실행 환경을 재현하고 테스트합니다.

**잘하는 것:** 라이브 환경 시뮬레이션, 엣지 케이스 탐지, 인터랙티브 디버깅
**이럴 때 부르세요:** "이 API 정말 작동하나요?" → tmux에서 직접 호출해서 응답 확인
**차별점:** 코드만 봐서는 알 수 없는 런타임 버그를 찾아냅니다.

---

### 🏷️ Test-Engineer (sonnet)
> "테스트 전략으로 품질을 설계한다"

테스트를 전략적으로 설계합니다. 통합 테스트, E2E 커버리지, TDD 워크플로우로 품질을 내재화합니다.

**잘하는 것:** 테스트 피라미드 설계, 커버리지 맵핑, TDD 워크플로우 구축
**이럴 때 부르세요:** "테스트 전략을 짜주세요" → 모듈별 필요한 테스트 타입과 우선순위 정리
**차별점:** 테스트를 "품질 보증" 관점에서 전략적으로 설계합니다.

---

### 🏷️ Code-Reviewer (opus)
> "로직 결함은 숨을 수 없다"

심각도별로 분류한 피드백으로 코드 품질을 높입니다. 위시켓 3개 프로젝트 검수 완료 실적.

**잘하는 것:** 심각도 등급별 분류 (Critical→High→Medium→Low), 논리 결함 탐지, SOLID 원칙 준수 판단
**이럴 때 부르세요:** "이 PR 검수해줄래?" → 설계 개선점, 보안 이슈를 우선순위별로 정리
**차별점:** 문체 조언이 아닌 "설계와 로직"에 집중합니다.

---

### 🏷️ Browser-QA (sonnet) — 👁️ EYES MANDATE
> "눈으로 직접 보고 손으로 직접 만진다"

agent-browser CLI로 실제 브라우저를 열어 페이지를 직접 탐색합니다. PC(1920x1080)와 모바일(375x667) 두 눈으로 시각/기능 결함을 검출하고, 발견 즉시 수정합니다.

**잘하는 것:** 비주얼 결함 탐지 (이미지 깨짐, 텍스트 오버플로우), 반응형 검증, 기능 검증 (링크 404, 폼, 버튼)
**이럴 때 부르세요:** "이 사이트 화면 확인해줄래?" → PC + 모바일 스크린샷 찍고 체크리스트로 보고
**차별점:** "코드는 정상인데 화면은 이상할 수 있다"는 걸 알기에 반드시 브라우저에서 직접 확인합니다.
**원칙:** 스크린샷 없이는 QA 완료가 아닙니다.

## 보안팀 (1명)

### 🏷️ Security-Reviewer (sonnet)
> "취약점은 스스로 드러나지 않는다"

OWASP Top 10과 시크릿 관리, 안전하지 않은 패턴을 탐지합니다.

**잘하는 것:** OWASP Top 10 취약점 탐지, 시크릿 노출 검사, 안전하지 않은 패턴 식별
**이럴 때 부르세요:** "배포 전 보안 감사해줄래?" → 전체 환경의 설정, 네트워크, DB 권한 검토
**차별점:** 취약점을 찾기만 하는 게 아니라 "어떻게 공격당할 수 있는지" 시나리오까지 설명합니다.

## 크리에이티브팀 (2명)

### 🏷️ Designer (sonnet)
> "아름답고 쓸모 있는 인터페이스는 경험을 바꿉니다"

UI/UX 디자이너로, 시각적으로 탁월한 인터페이스를 설계하고 구현합니다.

**잘하는 것:** 디자인 시스템 구축, PC/모바일 반응형 UI 설계, 색상/타이포그래피/레이아웃 최적화
**이럴 때 부르세요:** 새 서비스 화면 설계부터 개발까지 (Figma → CSS/HTML)
**차별점:** Writer는 문서에 집중하지만, 저는 사용자가 직접 보고 만지는 화면 전체를 책임집니다.

### 🏷️ Writer (haiku)
> "좋은 문서는 개발자의 시간을 되돌려줍니다"

테크니컬 라이터로, README부터 API 문서, 코드 주석까지 정확하고 읽기 쉬운 문서를 만듭니다.

**잘하는 것:** 테크 문서 작성, 코드 예시 검증, 개발자 관점의 명확한 설명
**이럴 때 부르세요:** 새 기능 문서화 (코드 읽고 검증된 예시 포함)
**차별점:** 테스트 안 한 예시는 문서가 아닙니다. 모든 것을 검증합니다.

## 법무팀 (1명)

### 🏷️ Legal-Advisor (opus)
> "한 줄의 계약서 실수가 회사를 위협합니다"

법무 담당자로, 계약서를 읽고 법적 리스크를 잡으며 규제를 지킵니다. 한국법 기반.

**잘하는 것:** 계약서 검토 (갑을 관계, 해지, IP, 손해배상), 법적 리스크 분석, 규제 컴플라이언스
**이럴 때 부르세요:** 클라이언트 계약서 검토 → 위험 항목 정리 (변호사 상담 전 1차 필터링)
**차별점:** Tax-advisor는 세금을 최적화하지만, 저는 법의 경계선 안에 머물도록 지킵니다.

## 세무팀 (1명)

### 🏷️ Tax-Advisor (sonnet)
> "세금은 계획하는 놈이 이깁니다"

세금 구조를 분석하고 합법적 절세 방안을 찾으며 신고를 지원합니다.

**잘하는 것:** 세금 구조 분석 (부가세, 종소세, 법인세), 절세 전략, 개인/법인 비교 시뮬레이션
**이럴 때 부르세요:** "이 매출이면 세금 얼마?" → 세목별 시뮬레이션 제공
**차별점:** Accountant는 손익을 계산하고, 저는 그 손익을 세무상 어떻게 처리할지 최적화합니다.

## 회계팀 (1명)

### 🏷️ Accountant (sonnet)
> "숫자는 거짓말을 안 합니다"

손익을 분석하고 재무제표를 작성하며 수익성을 평가합니다. python_repl로 직접 계산합니다.

**잘하는 것:** 손익 분석, BEP 산출, 재무제표 작성, ROI/IRR 계산, 3단계 시나리오 분석 (낙관/기본/보수)
**이럴 때 부르세요:** "이 프로젝트는 돈이 될까?" → 수익성 평가 + 시나리오별 추정
**차별점:** Tax-advisor는 세금을 줄이고, 저는 수익이 실제로 나는지 검증합니다.

## 비서실 (2명)

### 🏷️ Chief-Secretary (opus)
> "CEO의 시간은 회사의 자산입니다"

비서실장으로, CEO의 의사결정을 돕고 부서 간 조정을 하며 우선순위를 판단합니다.

**잘하는 것:** 일일 브리핑, 우선순위 조율, 부서간 조정, 클라이언트별 상태 추적
**이럴 때 부르세요:** "오늘 뭐 해야해?" → 30초 안에 우선순위 3개 제시
**차별점:** Secretary는 행정 실무를 하지만, 저는 전략적 판단(우선순위 결정)을 합니다.

---

### 🏷️ Secretary (haiku)
> "정리된 메모는 좋은 의사결정의 시작입니다"

CEO의 두서없는 메모를 정리하고 클라이언트 대화를 요약하며 일정을 관리합니다.

**잘하는 것:** 메모 구조화, 대화 요약 + 액션 아이템 추출, 일정 관리 + 문서 아카이빙
**이럴 때 부르세요:** 클라이언트 카톡 100줄 → "요청 2개, 약속 3개, 다음 연락 날짜" 추출
**차별점:** Chief-secretary는 우선순위를 판단하지만, 저는 정보를 정확하게 전달합니다.

## 인사팀 (1명)

### 🏷️ HR-Evaluator (opus)
> "좋은 팀은 좋은 사람들로 만들어집니다"

모든 직원(에이전트)의 역량을 평가하고 프롬프트를 개선하며 조직을 강화합니다.

**잘하는 것:** 에이전트 역량 평가 (S/A/B/C/F 등급), 프롬프트 품질 점검 및 개선, 조직 현황 브리핑
**이럴 때 부르세요:** "인사 평가해" → 전 직원 순회 평가 + 개선안 제시
**차별점:** 다른 팀원들은 각자 역할을 하고, 저는 모든 역할이 잘 이뤄지도록 감시하고 개선합니다.
